## 한 줄 포지셔닝

결제·주문·인증·외부 연동처럼 실패 비용이 큰 도메인에서 정합성, 상태 전이, 재처리, 운영 예외를 백엔드 구조로 풀어온 6년차 백엔드 엔지니어.

## 좋은 코드와 설계에 대한 생각

좋은 코드는 요구사항을 단순히 동작하게 만드는 코드가 아니라, 도메인의 변경이 어디까지 영향을 주는지 예측 가능하게 만드는 코드라고 생각합니다. 특히 결제·환불·정산처럼 실패 비용이 큰 영역에서는 가독성을 바탕으로 흐름이 명확해야 합니다. 어떤 상태에서 어떤 전이가 가능하고, 외부 API 실패·중복 요청·재시도·부분 성공이 발생했을 때 시스템이 어떤 보상 흐름을 타는지 코드만 봐도 추적 가능해야 합니다.

설계에서는 공통화를 무조건 많이 하는 것보다 경계를 잘 잡는 것이 중요하다고 생각합니다. 처음에는 재사용성을 위해 서비스를 넓게 공통화했지만, 도메인 서비스끼리 수평 참조가 늘어나면 작은 모델 변경도 여러 기능에 전파되고 테스트와 이관 순서가 복잡해진다는 점을 경험했습니다. 이후에는 정말 공통인 인프라·유틸·도메인 규칙만 공통 계층에 두고, 각 기능의 비즈니스 흐름은 응집도 있게 유지하는 방향을 선호합니다.

## 핵심 요약

Kubernetes 기반 모노레포 환경에서 결제, 주문, 환불, 정산, 제휴 연동, CMS 등 서비스 핵심 도메인을 개발·운영하고 있습니다. 요구사항을 기능 단위로만 보지 않고, 실제 비즈니스 플로우와 운영 과정에서 생길 수 있는 예외를 먼저 이해한 뒤 코드와 아키텍처에 녹여내는 데 강점이 있습니다.

이전에는 탈중앙화 생체인증 기술 기반의 무인 출입·결제 시스템을 개발했습니다. 기존 인증 흐름에서 민감한 생체 데이터의 전달 범위와 인증 순간의 네트워크 왕복이 커질 수 있는 문제를 발견하고, 데이터 비교 책임과 디바이스 경계를 재정의해 특허 등록된 키오스크 인증 플로우와 장치 연동 기반을 설계했습니다. 이 기술은 CES 2024 혁신상과 CES 2025 최고 혁신상을 수상했습니다.

기능 목록만 보고 바로 구현하기보다, 사용자와 운영자가 실제로 거치는 흐름을 먼저 이해하려고 합니다. 결제, 인증, 제휴 연동처럼 여러 상태와 예외가 얽힌 도메인에서는 비즈니스 흐름을 충분히 파악한 뒤 코드와 구조에 녹이는 편입니다. 익숙하지 않은 도메인도 빠르게 파고들어 문제를 정리하고, 필요한 역할이 비어 있으면 직접 메우며 일을 끝까지 가져가려고 합니다.

- **AI를 개발 산출물로 연결하는 사람**: AI 코딩 도구 사용을 넘어, 코드 리뷰 기준·테스트 시나리오·문서화 vault·LLM Agent 연동처럼 팀 생산성 문제를 직접 해결합니다.
- **일을 맡기면 불안하지 않은 사람**: 요구사항의 빈틈, 운영 예외, 사용자 흐름, 개발자 편의성까지 함께 보고 결과물을 더 섬세하게 만듭니다.

## AI & Developer Productivity

AI 도구를 단순 코드 생성 도구가 아니라, 레거시 이해·리뷰 품질·테스트 보강·문서화 비용을 줄이는 개발 산출물 제작 도구로 활용합니다.

- **레포지토리 지식화**: 여러 서비스 레포의 README, DB 스키마, 배포 방식, 주요 도메인 흐름을 구조화해 문서화 vault로 정리하고, LLM/AI Agent가 레포지토리 맥락을 잃지 않고 참조할 수 있는 컨텍스트 기반으로 활용했습니다.
- **리뷰 기준 표준화**: PR 변경 내용을 기준으로 사이드 이펙트, 누락된 테스트, 장애 가능성, 도메인 정합성 이슈를 점검하는 리뷰 체크리스트와 프롬프트를 만들었습니다.
- **레거시 분석 가속화**: Express.js 레거시 코드의 라우팅, 서비스 의존성, DB 접근 흐름을 AI로 요약하고, NestJS 이관 시 필요한 작업 단위와 리스크를 정리하는 방식으로 분석 시간을 줄였습니다.
- **테스트 시나리오 도출**: 복잡한 결제·환불·정산 로직에 대해 정상 케이스뿐 아니라 실패, 재시도, 중복 요청, 정합성 예외를 포함한 테스트 케이스 초안을 만들었습니다.

#### 주요 성과

- 신한/NH/KB/삼성증권 API를 연동하며, 계좌 개설·매매 기능에서 증권사별 응답 형식, 상태값, 예외 흐름의 차이를 백엔드에서 흡수해 일관된 사용자 플로우로 제공했습니다.
- 주식 계좌 개설 이벤트를 담당하며 외부 증권사 확인, 보상 지급, 사용자 알림이 한 요청에 묶이면 지연·실패·중복 지급 위험이 커진다고 판단했습니다. 이벤트 참여 조건 검증과 계좌 개설 상태 확인, 보상 지급 요청을 큐 기반 후속 처리로 분리했습니다.
- 보상 처리 결과를 SNS/앱 푸시로 비동기 안내하고, 실패·재시도 상황에서도 사용자가 이벤트 진행 상태를 확인할 수 있는 알림 흐름을 구성했습니다.
- 초기 Android 앱 MVP를 1인 개발해 백엔드 개발만 기다리던 출시 병목을 줄이고, 앱 출시 일정을 약 1개월 단축했습니다.
- Socket.IO + Redis 기반 실시간 주식 데이터 전달 구조의 구현·개선에 참여했습니다. 종목별 구독 단위로 필요한 데이터만 전달하고 Redis 기반 데이터 공유를 적용해 반응 속도 30% 개선, 서버 부하 40% 이상 절감에 기여했습니다.
- 커뮤니티 피드, 종목 태그, 댓글 기능을 주식 데이터와 연결해 사용자가 단순 게시글이 아니라 종목 중심으로 콘텐츠와 투자 정보를 탐색할 수 있는 흐름을 만들었습니다.
- AWS Lambda + EventBridge 기반 자동 종료와 Slack 운영 알림을 도입해 야간/비활성 시간대의 운영 비용과 장애 확인 부담을 줄였습니다.
- 서비스는 누적 다운로드 45만 건, 계좌 개설 4만 건을 달성했습니다.
